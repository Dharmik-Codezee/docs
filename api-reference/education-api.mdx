# Add Education for Jobseeker API

## Endpoint

`POST /api/jobseeker/education/add`

This endpoint is used to add education details for a jobseeker.

### Request

#### Headers

- `authorization`: Authorization token.

#### Body

- `education_type`: Numeric identifier for the type of education.
- `institute`: Name of the educational institute.
- `duration_from`: Start date of the education (formatted as a date).
- `duration_to`: End date of the education (formatted as a date).

### Validation

- `education_type` must not be empty and must be numeric.
- `institute` must not be empty.
- `duration_from` must not be empty and must be a valid date.
- `duration_to` must not be empty and must be a valid date.

### Response

- Status: 200 OK if successful, error status otherwise.
- Body: JSON response with relevant data.

### Notes

- Ensure to include the authorization token in the request header.
- The API supports adding education details with appropriate validation checks.
- Check the response for success or error messages.

### Examples

#### Request Example

```json
{
    "education_type": 2,
    "institute": "VNSG",
    "course": "B",
    "specialization": "Gujarat",
    "duration_from": "2020-06-15",
    "duration_to":"2023-04-10",
    "grading": "89"
}
```

`GET /api/jobseeker/education/list`

This endpoint is used to retrieve a list of education details for a jobseeker.

# View Education Details for Jobseeker API

## Endpoint

`GET /api/jobseeker/education/view/:education_id`

This endpoint is used to retrieve details of a specific education entry for a jobseeker.

### Request

#### Headers

- `authorization`: Authorization token.

#### Parameters

# Update Education for Jobseeker API

## Endpoint

`PATCH /api/jobseeker/education/update/:education_id`

This endpoint is used to update details of a specific education entry for a jobseeker.

### Request

#### Headers

- `authorization`: Authorization token.

#### Parameters

- `education_id` (URL parameter): The ID of the education entry to update. Should be a non-empty string with a minimum length of 24 characters.

#### Body

- `education_type`: Numeric value representing the type of education.
- `institute`: Name of the institute.
- `course`: Name of the course.
- `specialization`: Specialization in the course.
- `duration_from`: Start date of the education period.
- `duration_to`: End date of the education period.
- `grading` (optional): Grading information.

### Response

- Status: 200 OK if successful, error status otherwise.
- Body: JSON response with updated education details.

### Notes

- Ensure to include the authorization token in the request header.
- Provide valid values for education details in the request body.
- The API updates education details associated with the authenticated jobseeker.
- If the education entry is updated successfully, the updated details are returned in the response.
- If the education entry is not found, an error message is returned in the response.
- Check the response for success or error messages.

#### Request Example

```json
{
    "education_type": 1,
    "institute": "VNSGU",
    "course": "BA",
    "specialization": "Gujarati",
    "duration_from": "2020-06-15",
    "duration_to":"2023-04-10",
    "grading": "89"
}
```
