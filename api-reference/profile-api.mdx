# View Jobseeker Profile API

## Endpoint

`GET /api/jobseeker/profile/view`

This endpoint is used to view the profile of a jobseeker.

### Request

#### Headers

- `authorization`: Authorization token.

### Response

- Status: 200 OK if successful, error status otherwise.
- Body: JSON response containing the profile data of the jobseeker.

### Profile Data Structure

- `user_id`: The ID of the user.
- `country_code`: The country code of the user's phone number.
- `mobile_no`: The mobile number of the user.
- `user_email`: The email address of the user.
- `user_type`: The type of user (e.g., jobseeker).
- `language`: The language preference of the user.
- `first_name`: The first name of the user.
- `last_name`: The last name of the user.
- `full_name`: The full name of the user (concatenation of first name and last name).
- `date_of_birth`: The date of birth of the jobseeker.
- `about`: Information about the jobseeker.
- `gender`: The gender of the jobseeker.
- `location`: The location of the jobseeker.
- `work_position`: The current work position of the jobseeker.
- `work_status`: The work status of the jobseeker.
- `profile_img`: The profile image URL of the jobseeker.
- `identity_type`: The type of identity document (e.g., passport) of the jobseeker.
- `identity_card`: The identity card information of the jobseeker.
- `skills`: The skills of the jobseeker.
- `known_language`: The known languages of the jobseeker.
- `resume`: The resume of the jobseeker.
- `notification`: Notification settings of the jobseeker.
- `open_to_work`: Whether the jobseeker is open to work.
- `available_to_join`: Availability of the jobseeker to join.
- `employmentData`: Array containing employment history of the jobseeker.
- `educationData`: Array containing education history of the jobseeker.
- `certificateData`: Array containing certificate information of the jobseeker.

### Notes

- Ensure to include the authorization token in the request header.
- The profile data includes various details about the jobseeker such as personal information, work history, education history, and certificate information.
- If the profile data is successfully retrieved, it is returned in the response body.
- If the profile data is not found or there are errors, appropriate error messages are returned in the response.

# Update Jobseeker Profile API

## Endpoint

`PATCH /api/jobseeker/profile/update`

This endpoint is used to update the profile of a jobseeker.

### Request

#### Headers

- `authorization`: Authorization token.

#### Body

The request body should contain the following fields for updating the jobseeker's profile:

- `about`: Information about the jobseeker.
- `first_name`: The updated first name of the jobseeker.
- `last_name`: The updated last name of the jobseeker.
- `user_email`: The updated email address of the jobseeker.
- `country_code`: The updated country code of the user's phone number.
- `mobile_no`: The updated mobile number of the user.
- `date_of_birth`: The updated date of birth of the jobseeker.
- `gender`: The updated gender of the jobseeker.
- `location`: The updated location of the jobseeker.
- `work_status`: The updated work status of the jobseeker.
- `available_to_join`: The updated availability of the jobseeker to join.
- `known_language`: An array of updated known languages of the jobseeker.
- `skills`: An array of updated skills of the jobseeker.

### Response

- Status: 200 OK if successful, error status otherwise.
- Body: JSON response indicating whether the profile update was successful or not.

### Notes

- Ensure to include the authorization token in the request header.
- The request body should contain the fields to be updated in the jobseeker's profile.
- If the profile update is successful, a success message is returned in the response body.
- If there are any errors during the update process, appropriate error messages are returned in the response.
