# Add Certificate for Jobseeker API

## Endpoint

`POST /api/jobseeker/certificate/add`

This endpoint is used to add a new certificate for a jobseeker.

### Request

#### Headers

- `authorization`: Authorization token.

#### Body

- `certificate_name`: Name of the certificate.
- `file_url`: URL of the certificate file.

### Response

- Status: 200 OK if successful, error status otherwise.
- Body: JSON response with details of the added certificate.

### Notes

- Ensure to include the authorization token in the request header.
- Provide valid values for the certificate name and file URL in the request body.
- The API adds a new certificate entry associated with the authenticated jobseeker.
- If the certificate name already exists for the user, an error message is returned in the response.
- Check the response for success or error messages.

---

### Examples

#### Request Example

```json
{
    "certificate_name": "Degree_1",
    "file_url": "images/certificate.text"
}
```

### Response header
```json
{
    "status": 200,
    "err": 0,
    "data": {
        "user_id": "65bc8a08ee10fd14593b0ce5",
        "certificate_name": "Degree_1",
        "file_url": "images/certificate.text",
        "deleted": false,
        "created_by": "65bc8a08ee10fd14593b0ce5",
        "updated_by": "65bc8a08ee10fd14593b0ce5",
        "created_at": "2024-02-07T07:09:25.135Z",
        "updated_at": "2024-02-07T07:09:25.135Z",
        "_id": "65c32ca5bfe0f96757479320",
        "__v": 0
    },
    "msg": "Certificate added successfully"
}
```

If user not exist:

```json
{
   "status": 401,
   "err": 1,
   "data": {},
   "msg": "User not exist"
}
```

If same name already exist:
```json
{
    "status": 200,
    "err": 1,
    "data": {},
    "msg": "Certificate name already exist"
}
```

# View Certificate Details API

## Endpoint

`GET /api/jobseeker/certificate/view/:certificate_id`

This endpoint is used to view details of a specific certificate belonging to a jobseeker.

### Request

#### Headers

- `authorization`: Authorization token.

#### Parameters

- `certificate_id` (URL parameter): The ID of the certificate to view. Should be a non-empty string with a minimum length of 24 characters.

### Response

- Status: 200 OK if successful, error status otherwise.
- Body: JSON response containing the details of the certificate.

### Notes

- Ensure to include the authorization token in the request header.
- Provide the `certificate_id` parameter in the URL to specify the certificate to view.
- If the certificate is found, its details are returned in the response.
- If the certificate data is not found or if the user is unauthorized, appropriate error messages are returned in the response.

---

```json
{
    "status": 200,
```

# Update Certificate Details API

## Endpoint

`PATCH /api/jobseeker/certificate/update/:certificate_id`

This endpoint is used to update details of a specific certificate belonging to a jobseeker.

### Request

#### Headers

- `authorization`: Authorization token.

#### Parameters

- `certificate_id` (URL parameter): The ID of the certificate to update. Should be a non-empty string with a minimum length of 24 characters.

#### Body

- `certificate_name` (required): The name of the certificate.
- `file_url` (required): The URL of the certificate file.

### Response

- Status: 200 OK if successful, error status otherwise.
- Body: JSON response containing the updated details of the certificate.

### Notes

- Ensure to include the authorization token in the request header.
- Provide the `certificate_id` parameter in the URL to specify the certificate to update.
- Include the `certificate_name` and `file_url` fields in the request body to update the certificate details.
- If the certificate is successfully updated, the updated details are returned in the response.
- If the certificate data is not found, the user is unauthorized, or there are validation errors, appropriate error messages are returned in the response.

---

```json
{
    "certificate_name": "Degree",
    "file_url": "images/certificate.text"
}
```
