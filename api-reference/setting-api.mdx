# Setting Page API

## Endpoint

`GET /api/setting/:type`

This endpoint retrieves the setting page data based on the specified type.

### Request

- Headers
  - `authorization`: Authorization token for the authenticated user.
- Parameters
  - `type`: Specifies the type of setting page to retrieve.

### Response

- Status: 
  - 200 OK if the operation is successful.
  - Error status codes for failures.
- Body: JSON response containing the setting page data.
  - The response includes the HTML content of the setting page for the specified type.
  - Error messages are returned if the authenticated user does not exist or if the setting page data for the specified type is not found.

### Notes

- This endpoint requires authentication via the `authorization` header.
- The `type` parameter specifies the type of setting page to retrieve.
- The response includes the HTML content of the setting page for the specified type.
- Error messages are returned if the authenticated user does not exist or if the setting page data for the specified type is not found.

# Notification Status API

## Endpoint

`POST /api/setting/notification/:status`

This endpoint updates the notification status for the user or business.

### Request

- Headers
  - `authorization`: Authorization token for the authenticated user.
- Parameters
  - `status`: Specifies the notification status. Should be a boolean value.

### Response

- Status: 
  - 200 OK if the operation is successful.
  - Error status codes for failures.
- Body: JSON response indicating the status of the notification update.
  - If the update is successful, a success message is returned.
  - If the update fails, an error message is returned.

### Notes

- This endpoint requires authentication via the `authorization` header.
- The `status` parameter specifies the new notification status, which should be a boolean value.
- If the user is a job seeker, the notification status is updated for the job seeker.
- If the user is a business member, the notification status is updated for the business.
- Error messages are returned for various scenarios such as user not existing, business not existing, user not being a member of the business, etc.
