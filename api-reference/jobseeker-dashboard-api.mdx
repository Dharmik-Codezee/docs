# Jobseeker Dashboard Count API

## Endpoint

`GET /api/jobseeker/dashboard-count`

This endpoint retrieves dashboard count data for the authenticated jobseeker user.

### Request

- Headers
  - `authorization`: Authorization token for the authenticated user.

### Response

- Status: 200 OK if the operation is successful, error status otherwise.
- Body: JSON response containing the jobseeker dashboard count data.
  - The response includes the jobseeker's profile completion percentage and the count of unread notifications.
  - If the authenticated user is not a jobseeker or does not exist, appropriate error messages are returned.
  - Proper error handling is implemented to handle scenarios such as unauthorized access, invalid input, or database errors.

### Notes

- This endpoint requires authentication via the `authorization` header.
- The response includes the jobseeker's profile completion percentage and the count of unread notifications.
- The profile completion percentage indicates how much of the jobseeker's profile has been completed.
- The notification count represents the number of unread notifications for the jobseeker.
- Error messages are returned if the authenticated user is not a jobseeker or if an error occurs during data retrieval.
- The endpoint ensures that only jobseeker users can access the dashboard count data.

---
### Success Response (200 OK)

```json
{
    "status": 200,
    "err": 0,
    "data": {
        "profile_percent": 100,
        "notification_count": 0
    },
    "msg": "Please find dashboard data."
}
```

### Error Response (User Not Exist)

```json
{
  "status": 401,
  "err": 1,
  "data": {},
  "msg": "User not exist"
}
```





# Jobseeker Dashboard Banner API

## Endpoint

`GET /api/jobseeker/dashboard-banner`

This endpoint retrieves banner data for the jobseeker dashboard.

### Request

- Headers
  - `authorization`: Authorization token for the authenticated user.

### Response

- Status: 
  - 200 OK if the operation is successful.
  - Error status codes for failures.
- Body: JSON response containing the banner data for the jobseeker dashboard.
  - The response includes an array of banners sorted by creation date in descending order.
  - If banners are found, they are randomly sampled to return a maximum of 5 banners.
  - If no banners are found, an empty array is returned.
  - Error handling is implemented to handle scenarios such as unauthorized access, database errors, or no banners found.

### Notes

- This endpoint requires authentication via the `authorization` header.
- The response includes banner data to be displayed on the jobseeker dashboard.
- Banners are sorted by creation date in descending order to prioritize newer banners.
- A maximum of 5 banners are randomly sampled from the available banners.
- Error messages are returned if the authenticated user is not a jobseeker or if an error occurs during data retrieval.
